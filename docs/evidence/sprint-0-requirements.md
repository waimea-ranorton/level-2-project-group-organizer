# Project Requirements

## Identified Problem or Need

Often group projects become hectic, files and documents become scrambled, people forget what parts they need to do and when they need to do them by, everyone ends up with their own individual work that needs combining and organizing, etc.

## End-User Requirements

My client is a student, which is where the problem came from, a need for organizing class projects, however I could imagine this app also being used in workplaces.
These end-users need the app to be: Quick to use, Live updating, Accessible Useable, etc.

## Proposed Solution

My app aims to organize these groups, providing a base for their project, here members of the group can: Add and access each related file, Attach people to files (say Alex is attached to the powerpoint and Steve and Steven are working on the word document), Add notes regarding each file or the project as a whole, Add roles under each person, etc.

# Relevant Implications

## USABILITY --------------------------------------------------

Usability means to be able to be picked up and immediately and be able to be used effectively and easily.

### Relevance to the System

This app needs to be "Quick to use" as I mentioned earlier, on top of this, if the app were not useable, it would frustrate and confuse users which would lead them not to use it, for this reason usability is also very important as it ties in with accessibility.

### Impact / Considerations

I plan to make my site easy to use by: Keeping things such as navigation and buttons, fonts and colors consistent throughout the whole app, sticking to the general layout for apps/websites (heading at top, nav at top, main body in middle, footer at bottom, ECT), showing clear feedback to users (when a document is added, small pop-up saying so, when a button is pressed it changes colour, etc.)

## END USER --------------------------------------------------

The implication of an end user means to consider that the app is going to be used by real/other people, and not just going by what you think is good.

### Relevance to the System

This app needs to be catered to the specific users that will be using it, if the app was just based off of what I think works, and is good, then there is poetical that it dose not actually work that well for end users.

### Impact / Considerations

Moving forward I plan to meet my end-users needs I firstly need to considered any feedback and requests from them, in initial conversation they mentioned that the app should "Not be too busy to look at" meaning they want a simplistic, minimalistic design with nitty gritty features tucked away.

Secondly I need to consider a wider group, how any students would interact with this app, this could mean things like: Writing content in simple language so users of different reading levels can understand, priority over making common/main task quick and easy to complete, using symbols for key areas (home = house icon) so users can navigate quickly, etc.

## FUNCTIONALITY------------------------------------------------

Functionality refers to how well something works.

### Relevance to the System

As mentioned earlier, bearing in mind this app is intended for students, most of which are both busy and with short attention span, it is very important that the app functions well, if it breaks, crashes, or runs slowly this will frustrate the user and effects the perceived professionally and reliability of the app.

### Impact / Considerations

I intent to optimize functionality by: Testing each button, form, link, post etc before publishing updates, checking how the app runs across different devices (mobile, tablet, desktop), etc.

## ACCESSIBILITY------------------------------------------------

Acessibilit means for somthing to be able to be used by the widest range of people possible.

### Relevance to the System

"Students" as intended users is a very broad catogory, this can include people of all ranges of ability, this could include things like color-blindness, dyslexia, impaired vision, ect.

### Impact / Considerations

I aim to make my app acessible to as many users as possible through implementing things such as: Having generic HTML tags (header, main, nav, section, etc), adding 'alt' text to any images to a screen-reader can read it out, having good contrast of values.
